⚡ Activist Terminal

Production-grade Shareholder Activism Analytics Platform

Next.js 15 FastAPI PostgreSQL Prisma Event Studies

🎯 What This Platform Answers

  • Which activist investors are currently active?
  • Which companies are being targeted and what demands are activists making?
  • Which campaigns escalated to proxy fights?
  • How many board seats were won, settled, or lost?
  • How did stocks perform before campaigns, around announcements, and afterward?
  • Which activists have the best hit rates by campaign type, sector, or outcome?

📊 Event-Study Methodology

Proper finance methodology — not naive price deltas:

  • Market model: OLS regression with estimation window [-280, -30] trading days vs SPY
  • Minimum 150 observations required for statistical validity
  • CAR & BHAR: Cumulative Abnormal Returns and Buy-and-Hold Abnormal Returns
  • Trading-day offsets (not calendar days) for accurate windows
  • Materialized metrics keyed by (campaign, anchor event, benchmark, method, window)
  • Incremental recomputation gated by inputs hash

🏛️ Architecture Highlights

Schema Split

Prisma owns app schema; Alembic manages staging schema

Event Sourcing

Entity Resolution

CIK → CUSIP → alias exact → trigram fuzzy cascade with human review queue

21-Table Schema

investors, campaigns, events, demands, proxy_fights, price_metrics, and more
